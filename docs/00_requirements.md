# 家計管理システム 要件定義書 Ver1.0

## 1. プロジェクト概要

### 目的

現行のGoogleフォーム＋Googleスプレッドシートによる家計簿を全面刷新し、入力性・保守性・拡張性を向上させたWebアプリケーションを開発する。

### コンセプト

> **30秒で入力し、5分で振り返る。10年以上使える家計簿。**

---

## 2. 開発方針

### 優先順位

```
① 入力の速さ
② シンプルさ
③ 保守性
④ 拡張性
⑤ 見た目
```

**重要**: Excelの延長ではなく、専用アプリとして設計する。

---

## 3. システム構成

```
iPhone / PC / iPad
      │
PWA (HTML/CSS/JavaScript)
      │
Google Apps Script
      │
Google Spreadsheet (Database)
```

- サーバーは使用しない
- 完全無料で運用する

## 6. 入力画面

### 必須項目

- 収入／支出
- 日付
- 金額
- カテゴリ

### 任意項目

- 店舗
- タグ
- メモ

---

### 日付

- 初期値は当日
- 変更後は保持する
- まとめ入力を前提とする

### 金額

- 常に正数入力
- 収入・支出で符号を判断
- 日本円のみ
- 税込入力のみ

### カテゴリ

- 2階層構造
- 例:
  ```
  食費
   ├ スーパー
   ├ コンビニ
   ├ 外食
  ```
- 自由追加・編集可能

### 店舗

- 任意入力
- 履歴候補を表示
- 役割: レシートを後から特定しやすくするため
- 分析の主目的ではない

例:
```
カテゴリ: スーパー
店舗: オーケー
```

### タグ

- 自由入力
- 複数登録可能
- 用途: イベント・横断分析
- 例:
  ```
  旅行
  帰省
  プレゼント
  Amazonセール
  ```

**注意**: 店舗用途には使用しない。

### メモ

- 任意入力
- 通常は折りたたみ表示

## 8. 集計

### 対応

- マスタ管理

- 重複登録防止

---

## 10. 資産管理

- 家計簿とは連携しない
- 月末残高のみ入力
- 対象:
  ```
  銀行
  証券
  現金
  ローン
  ```
- 推移のみ表示する

## 12. データ設計

Google SpreadsheetをDBとして利用。

### テーブル

```
Transactions      # トランザクション
Categories        # カテゴリマスタ
Tags              # タグマスタ
Stores            # 店舗マスタ
RecurringRules    # 定期収支ルール
Accounts          # 資産口座マスタ
AssetSnapshots    # 資産残高スナップショット
Settings          # システム設定
Logs              # 操作ログ
```

**重要**:
- 表示専用シートは作成しない
- 集計はJavaScriptで実施

---

## 13. データ仕様

- 全データにUUIDを付与
- 論理削除を採用
- 作成日時・更新日時を保持

---

## 14. 非機能要件

- 完全無料
- 高速
- PWA対応
- ホーム画面追加可能
- 永年保存
- 保守しやすい構造
- 自動保存
- 通信エラー時リトライ

---

## 15. UI/UX方針

- 入力速度最優先
- 必須項目は最小限
- 設定画面以外で設定変更を行わない
- 片手操作を基本とする
- 円グラフなど不要な情報は表示しない

---

## 16. 運用

入力するだけ
```

② 資産残高入力
③ 終了
```

---

## 20. 設計原則

1. **入力を最優先する**
2. **必須項目を増やさない**
3. **タグで表現できるものはタグで管理する**
4. **スプレッドシートはDBであり、UIはアプリに集約する**
5. **集計データは保存せず、その都度計算する**
6. **長期間運用しても保守しやすい構造を維持する**
